### 3.5 关键设计决策

1. **为什么编码器用软件EXTI而不是硬件TIM编码器模式？**
   - PA11/PA12在STM32F030K6T6上没有定时器通道映射，无法用硬件编码器模式
   - 只能用软件EXTI双边沿触发+电平比较判断方向

2. **为什么PA8作为EC11公共端？**
   - EC11的C脚需要接地，原理图没有直接接地而是接到PA8
   - PA8配置为开漏输出低电平，模拟GND
   - 开漏输出不怕外部短路到地

3. **为什么PB0按键用轮询而不是EXTI？**
   - PB0和PA11/PA12不在同一EXTI线上（PB0是EXTI0，PA11/PA12是EXTI4_15），可以都用EXTI
   - 但按键需要消抖（20ms）和长按检测（2s），用1ms定时器轮询更方便实现状态机
   - 所以PA11/PA12用EXTI检测旋转，PB0用TIM3轮询检测按键

4. **为什么PWM用TIM17而不是TIM3？**
   - TIM3被1ms定时器占用（ARR=47999用于1ms中断）
   - TIM17的CH1可以映射到PA7(AF5)，正好是加热器MOSFET栅极

5. **为什么PB6/PB7用开漏输出？**
   - I2C协议要求开漏+外部上拉，推挽会导致SDA线短路

### 4.1 user_code/encoder.h


### 4.2 user_code/encoder.c

**旋转检测**：EXTI双边沿触发 → `enc_exti_callback()` → 比较A/B电平判断方向 → 累加 `enc_delta`

**按键检测**：TIM3(1ms)中断 → `enc_timer_tick()` → 4状态消抖状态机：
- IDLE → DEBOUNCE(20ms) → PRESSED → RELEASE_DEBOUNCE(20ms) → IDLE
- 长按≥2s触发 `btn_long_flag`
- 短按（释放时未达2s）触发 `btn_short_flag`

**API**：
- `enc_get_delta()` — 读取并清零旋转增量（正=顺时针，负=逆时针）
- `enc_button_pressed()` — 读取并清零短按标志
- `enc_button_long_press()` — 读取并清零长按标志

### 4.3 main.c 修改点

| 区域 | 内容 |
|------|------|
| Includes | `#include "encoder.h"` |
| USER CODE 2 | `enc_init()` + `HAL_TIM_Base_Start_IT(&htim3)` + `HAL_TIM_PWM_Start(&htim17, ...)` |

## 六、踩坑记录

### 6.1 EC11引脚顺序（ACB而非ABC）

EC11常见规格为顺序"A-C-B"，而非直觉上的"A-B-C"。接错后B相始终为0，导致方向判断交替±1。正确接线应为：

```
EC11引脚:  A  C  B
连接:     PA11 PA8 PA12
```

### 6.2 EXTI 对比 定时器轮询

两种方案均可工作，最终采用EXTI方案：
- **EXTI方案**：双边沿触发，响应快，四倍频解码（每格±4）
- **定时器轮询方案**：1ms采样查表，天然过滤机械抖动，每格±4

两种方案在正确接线后都能正常工作。

### 6.3 GPIO NoPull的上拉问题

PA11/PA12在GPIO_Input模式下必须配置Pull-up，否则A/B断开时引脚浮空读数随机。在EXTI模式下NoPull即可（EC11内部有上拉回路）。
