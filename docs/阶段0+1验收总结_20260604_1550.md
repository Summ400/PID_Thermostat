### 1.2 时钟配置（最终方案）

```
HSI(8MHz) → PLL Source: HSI÷2
          → PREDIV: /1
          → PLLMUL: ×12
          → System Clock: PLLCLK
          → HCLK = 48MHz
          → APB1 = 48MHz
```

> ⚠️ HSI=8MHz，STM32F030的PLL输入要求≤1MHz，必须HSI÷2→4MHz后再PLL×12→48MHz。原设计使用HSE(8MHz) → PLL×6 → 48MHz，因HSE不起振改用HSI

### 1.3 遇到的问题与解决

#### 问题1：串口无输出

- **现象**：烧录后串口助手收不到任何数据
- **排查过程**：
  1. 确认 MicroLIB 已启用
  2. 添加 `HAL_UART_Transmit` 直接发送单字节测试 → 仍无输出
  3. 判断程序可能卡在 `SystemClock_Config()` 中的 `Error_Handler()`
- **根因**：HSE外部晶振不起振，`HAL_RCC_OscConfig()` 返回错误，进入死循环
- **解决**：改用 HSI 内部时钟

#### 问题2：HSE外部晶振不起振

- **晶振型号**：X50328MSB2GI (YSX530GA系列, 8MHz, 负载电容20pF, SMD5032-2P)
- **外部电容**：C1=C2=22pF
- **负载电容计算**：
  ```
  CL = (C1×C2)/(C1+C2) + Cs = (22×22)/(22+22) + 5 = 16pF
  ```
  晶振要求 CL=20pF，实际只有16pF，偏小4pF
- **正确匹配**：C1=C2=30pF → CL = 15+5 = 20pF
- **当前方案**：暂用HSI继续开发，后续更换30pF电容或低ESR晶振

### 1.4 验收结果

- ✅ 工程编译 0 Error, 0 Warning
- ✅ 串口助手稳定收到打印信息
- ✅ MicroLIB + printf重定向正常工作

### 2.3 实现的函数

| 函数 | 功能 |
|------|------|

| `temp_read()` | 读取16位数据，返回摄氏度(float) |
| `temp_thermocouple_ok()` | 检查热电偶是否断线 |

### 2.4 MAX6675通信协议

```
CS拉低 → 发送16个SCK时钟 → 读取MISO数据 → CS拉高

16位数据格式：
D15      : Dummy
D14~D3   : 12位温度值 (0.25°C/LSB)
D2       : 热电偶断线标志 (1=断线)
D1~D0    : Dummy

温度 = (data >> 3) × 0.25
```

### 2.5 引脚连接

```
MAX6675_CS   → PA4 (GPIO_Output, Push-Pull, High, 初始High)
MAX6675_SCK  → PA5 (GPIO_Output, Push-Pull, High, 初始Low)
MAX6675_MISO → PA6 (GPIO_Input, NoPull)
MAX6675_VCC  → 3V3
MAX6675_GND  → GND
```

### 2.6 遇到的问题与解决

#### 问题：温度读数为0.00°C

- **现象**：串口输出 `Temp: 0.00 C | TC: OK`，热电偶检测正常但温度为0
- **根因**：CubeMX中PA4/PA5/PA6未配置GPIO，`MX_GPIO_Init()` 为空
- **解决**：在CubeMX中配置PA4(Output)、PA5(Output)、PA6(Input)后重新生成代码

- ✅ 手摸热电偶温度上升
- ✅ 拔掉热电偶显示 TC: OPEN!
- ✅ 每500ms读取一次（MAX6675转换时间约170ms，500ms间隔安全）
